<task type="auto">
  <name>Task 2: Build UsersView with DataTable, loading, delete confirmation, and self-delete prevention</name>
  <files>simple-admin-spa/src/views/UsersView.vue</files>
  <action>
Replace the placeholder UsersView.vue with a full implementation using Vue 3 Composition API (`<script setup lang="ts">`):

**Data fetching:**
- `users` ref of type `UserListDto[]`, initially empty
- `loading` ref boolean, initially `false`
- `currentUserEmail` ref string or null, initially `null`
- On mount (`onMounted`): call `fetchUsers()` and `fetchCurrentUser()` in parallel
- `fetchUsers()`: set `loading = true`, call `getApiUsers()`, set `loading = false`. On error, show toast: `toast.add({ severity: 'error', summary: 'Error', detail: (error as any)?.detail ?? 'Failed to load users', life: 3000 })`. On success, set `users.value = data ?? []`.
- `fetchCurrentUser()`: call `getApiMe()`, cast result as `{ sub?: string; email?: string }`, store email in `currentUserEmail`. Silently ignore errors (non-critical).

**Template layout:**
- Page heading: `<h1>Users</h1>`
- "Create User" button above the table, right-aligned, using `<div class="flex justify-end mb-4">`. Button uses `icon="pi pi-plus"` label="Create User", `@click="router.push('/users/new')"`.
- `<DataTable :value="users" :loading="loading">` with two columns:
  - `<Column field="email" header="Email" />`
  - `<Column header="Actions">` with `#body="{ data: row }"` template slot containing:
    - Edit button: `<Button icon="pi pi-pencil" severity="info" text rounded @click="router.push('/users/' + row.id + '/edit')" />`
    - Delete button: `<Button icon="pi pi-trash" severity="danger" text rounded :disabled="row.email === currentUserEmail" @click="confirmDelete(row)" />`

**Delete confirmation:**
- `confirmDelete(user: UserListDto)`: call `confirm.require({ message: 'Are you sure you want to delete this user?', header: 'Delete User', icon: 'pi pi-exclamation-triangle', rejectLabel: 'Cancel', acceptLabel: 'Delete', acceptClass: 'p-button-danger', accept: async () => { ... } })`
- In the `accept` callback: call `deleteApiUsersById({ path: { id: user.id } })`. On error, show error toast. On success, show success toast "User deleted" and remove the user from `users.value` locally (filter by id) — no need to re-fetch the list.

**Imports needed:**
- `ref, onMounted` from vue
- `useRouter` from vue-router
- `DataTable` from primevue/datatable
- `Column` from primevue/column
- `Button` from primevue/button
- `useToast` from primevue/usetoast
- `useConfirm` from primevue/useconfirm
- `getApiUsers, getApiMe, deleteApiUsersById` from @/client/sdk.gen
- `UserListDto` from @/client/types.gen

**No scoped styles needed** — PrimeVue handles all component styling.
