  <action>
1. Create `SimpleAdmin.Api/Controllers/AuthorizationController.cs`:

   Implement the passthrough Authorization Code flow controller exactly as documented in the research:

   - `[ApiController]` attribute, no route prefix (uses `~/connect/` routes)
   - `Authorize()` method handling both `[HttpGet("~/connect/authorize")]` and `[HttpPost("~/connect/authorize")]`:
     a. Get the OpenIddict server request from `HttpContext.GetOpenIddictServerRequest()`
     b. Authenticate against `CookieAuthenticationDefaults.AuthenticationScheme`
     c. If not authenticated, return `Challenge` with cookie scheme, setting `RedirectUri` to the current request path + query string
     d. If authenticated, build a `ClaimsIdentity` with `TokenValidationParameters.DefaultAuthenticationType` as authentication type, `Claims.Name` as name type, `Claims.Role` as role type
     e. Set `Claims.Subject` (from `ClaimTypes.NameIdentifier`), `Claims.Email` (from `ClaimTypes.Email`), `Claims.Name` (from `ClaimTypes.Name`)
     f. Create `ClaimsPrincipal`, call `SetScopes(request.GetScopes())`, return `SignIn` with `OpenIddictServerAspNetCoreDefaults.AuthenticationScheme`
   - `Exchange()` method `[HttpPost("~/connect/token")]`:
     a. Get the OpenIddict server request
     b. If `IsAuthorizationCodeGrantType()`, authenticate with `OpenIddictServerAspNetCoreDefaults.AuthenticationScheme` and return `SignIn` with the principal
     c. Otherwise throw `InvalidOperationException("Unsupported grant type.")`

   Required usings: `Microsoft.AspNetCore.Authentication`, `Microsoft.AspNetCore.Authentication.Cookies`, `Microsoft.AspNetCore.Mvc`, `Microsoft.IdentityModel.Tokens`, `OpenIddict.Abstractions`, `OpenIddict.Server.AspNetCore`, `System.Security.Claims`

2. Create `SimpleAdmin.Api/Controllers/ApiController.cs`:

   A simple protected endpoint:
   ```csharp
   [ApiController]
   [Route("api")]
   public class ApiController : ControllerBase
   {
       [HttpGet("me")]
       [Authorize(AuthenticationSchemes = OpenIddictValidationAspNetCoreDefaults.AuthenticationScheme)]
       public IActionResult Me()
       {
           var sub = User.FindFirstValue(OpenIddict.Abstractions.Claims.Subject);
           var email = User.FindFirstValue(OpenIddict.Abstractions.Claims.Email);
           return Ok(new { sub, email });
       }
   }
   ```

   Required usings: `Microsoft.AspNetCore.Authorization`, `Microsoft.AspNetCore.Mvc`, `OpenIddict.Validation.AspNetCore`, `System.Security.Claims`

2. Create `SimpleAdmin.Api/Pages/Account/Login.cshtml.cs` (PageModel):

   ```csharp
   public class LoginModel : PageModel
   {
       private readonly SignInManager<ApplicationUser> _signInManager;
       public LoginModel(SignInManager<ApplicationUser> signInManager)
           => _signInManager = signInManager;

       [BindProperty] public string Email { get; set; } = "";
       [BindProperty] public string Password { get; set; } = "";
       [BindProperty(SupportsGet = true)] public string? ReturnUrl { get; set; }
       public string? ErrorMessage { get; set; }

       public void OnGet(string? returnUrl = null) => ReturnUrl = returnUrl;

       public async Task<IActionResult> OnPostAsync(string? returnUrl = null)
       {
           returnUrl ??= ReturnUrl;
           var result = await _signInManager.PasswordSignInAsync(
               Email, Password, isPersistent: false, lockoutOnFailure: false);

           if (!result.Succeeded)
           {
               ErrorMessage = "Invalid email or password";
               ReturnUrl = returnUrl;
               return Page();
           }

           return Redirect(returnUrl ?? "/");
       }
   }
   ```

   Required usings: `Microsoft.AspNetCore.Identity`, `Microsoft.AspNetCore.Mvc`, `Microsoft.AspNetCore.Mvc.RazorPages`, `SimpleAdmin.Api.Models`
