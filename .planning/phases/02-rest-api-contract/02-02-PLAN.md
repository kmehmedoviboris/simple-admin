  <name>Task 1: Extract TokenHelper from AuthFlowSmokeTests for reusable token acquisition</name>
  <files>SimpleAdmin.Tests/Helpers/TokenHelper.cs</files>
  <action>
Create `SimpleAdmin.Tests/Helpers/TokenHelper.cs` with a static class `TokenHelper` in namespace `SimpleAdmin.Tests.Helpers`.

Extract the PKCE token acquisition logic from `AuthFlowSmokeTests.FullAuthCodeFlow_ReturnsAccessToken` into a static async method:

```csharp
public static async Task<string> GetAccessTokenAsync(HttpClient client, string email = "admin@simpleadmin.local", string password = "Admin1234!")
```

The method should:
1. Generate PKCE code_verifier and code_challenge (copy the `GeneratePkce()` logic from AuthFlowSmokeTests)
2. Build the authorize URL with response_type=code, client_id=simple-admin-spa, redirect_uri=http://localhost:5173/callback, scope=openid email profile, code_challenge, code_challenge_method=S256, state=test_state
3. GET the authorize URL (expect 302 to /Account/Login)
4. GET the login page, extract __RequestVerificationToken via regex
5. Extract ReturnUrl from the login URL query string
6. POST the login form with email, password, ReturnUrl, and __RequestVerificationToken
7. Follow redirect chain until reaching http://localhost:5173/callback?code=...
8. Extract the authorization code from the callback URL
9. POST /connect/token with grant_type=authorization_code, code, redirect_uri, client_id, code_verifier
10. Parse the JSON response and return the access_token string

Throw `InvalidOperationException` if any step fails (no code received, no token returned).

The caller is responsible for creating the HttpClient with `AllowAutoRedirect = false` and `HandleCookies = true` from `TestWebApplicationFactory`. The helper should NOT create or configure the client — it receives it as a parameter.

Required usings: `System.Security.Cryptography`, `System.Text`, `System.Text.RegularExpressions`, `System.Net`, `System.Text.Json`, `Microsoft.AspNetCore.WebUtilities`.

After creating TokenHelper, do NOT refactor AuthFlowSmokeTests to use it — that is out of scope for this plan (avoid touching existing passing tests).
