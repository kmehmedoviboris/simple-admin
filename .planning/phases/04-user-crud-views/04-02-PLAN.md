<task type="auto">
  <name>Task 1: Create UserForm.vue shared component with inline validation</name>
  <files>simple-admin-spa/src/components/UserForm.vue</files>
  <action>
Create `UserForm.vue` using Vue 3 Composition API (`<script setup lang="ts">`).

**Props interface:**
```typescript
interface Props {
  initialEmail?: string    // empty string for create, pre-filled for edit
  initialPassword?: string // always empty string
  isCreate: boolean        // true = create mode, false = edit mode
  loading: boolean         // controls submit button :loading state
}
```
Use `withDefaults(defineProps<Props>(), { initialEmail: '', initialPassword: '' })`.

**Emits:**
```typescript
defineEmits<{
  submit: [value: { email: string; password: string }]
}>()
```

**Local state:**
- `email` ref initialized from `props.initialEmail`
- `password` ref initialized from `props.initialPassword` (always '')
- `errors` ref of type `Record<string, string>`, initially `{}`

**Validation function** (`validate(): boolean`):
- Clear `errors.value = {}`
- If `!email.value.trim()`: set `errors.value.email = 'Email is required'`
- Else if email does not match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`: set `errors.value.email = 'Enter a valid email address'`
- If `props.isCreate && !password.value`: set `errors.value.password = 'Password is required'`
- Return `Object.keys(errors.value).length === 0`

**Submit handler** (`onSubmit()`):
- If `!validate()` return early (no API call)
- Emit `'submit'` with `{ email: email.value, password: password.value }`

**Watch for initialEmail changes** — when the parent loads user data asynchronously (edit mode), the email ref must update. Use `watch(() => props.initialEmail, (val) => { email.value = val ?? '' })`.

**Template:**
- Wrap in a `<form @submit.prevent="onSubmit">` — prevents default form submission
- Page heading passed via slot or simply not included (parent views add their own heading)
- Email field:
  ```html
  <div class="flex flex-col gap-2 mb-4">
    <label for="email">Email</label>
    <InputText id="email" v-model="email" :invalid="!!errors.email" fluid />
    <Message v-if="errors.email" severity="error" size="small" variant="simple">{{ errors.email }}</Message>
  </div>
  ```
- Password field:
  ```html
  <div class="flex flex-col gap-2 mb-4">
    <label for="password">Password{{ isCreate ? '' : ' (leave blank to keep current)' }}</label>
    <Password id="password" v-model="password" :invalid="!!errors.password" :feedback="false" fluid toggleMask />
    <Message v-if="errors.password" severity="error" size="small" variant="simple">{{ errors.password }}</Message>
  </div>
  ```
- Submit button and back link:
  ```html
  <div class="flex gap-2 mt-4">
    <Button :label="isCreate ? 'Create' : 'Save'" type="submit" :loading="loading" />
    <Button label="Cancel" severity="secondary" text @click="router.push('/users')" />
  </div>
  ```

**Imports:**
- `ref, watch` from vue
- `useRouter` from vue-router
- `InputText` from primevue/inputtext
- `Password` from primevue/password
- `Button` from primevue/button
- `Message` from primevue/message

**Important:** `:feedback="false"` on Password to suppress the strength meter popup. `toggleMask` to allow showing/hiding password. `fluid` on inputs for full-width layout. Use `:loading` alone on submit button — do NOT also bind `:disabled` (PrimeVue Button loading conflict per research pitfall 7).

  <name>Task 2: Create UserCreateView and UserEditView wired to API with toast feedback</name>

  <action>
**UserCreateView.vue** (`<script setup lang="ts">`):
- Page heading: `<h1>Create User</h1>`
- `loading` ref boolean, initially `false`
- Import `useToast` from primevue/usetoast, `useRouter` from vue-router
- Import `postApiUsers` from @/client/sdk.gen
- Import `UserForm` from @/components/UserForm.vue
- `onSubmit` handler receives `{ email, password }`:
  - Set `loading.value = true`
  - Call `postApiUsers({ body: { email, password } })`
  - Set `loading.value = false`
  - If `error`: `toast.add({ severity: 'error', summary: 'Error', detail: (error as any)?.detail ?? 'Failed to create user', life: 3000 })` and return
  - On success: `toast.add({ severity: 'success', summary: 'Created', detail: 'User created', life: 3000 })` then `router.push('/users')`
- Template:
  ```html
  <div class="p-4">
    <h1>Create User</h1>
    <UserForm :is-create="true" :loading="loading" @submit="onSubmit" />
  </div>
  ```

**UserEditView.vue** (`<script setup lang="ts">`):
- Page heading: `<h1>Edit User</h1>`
- `loading` ref boolean (form submit), `pageLoading` ref boolean (initial data fetch)
- `userEmail` ref string, initially `''`
- `userId` from `useRoute().params.id as string`
- Import `useToast`, `useRouter`, `useRoute` from vue-router
- Import `getApiUsers, putApiUsersById` from @/client/sdk.gen
- Import `UserForm` from @/components/UserForm.vue

- **On mount** (`onMounted`): set `pageLoading.value = true`, call `getApiUsers()` to get all users, find the user with matching `id`, set `userEmail.value = foundUser.email`. Set `pageLoading.value = false`. If user not found or error: show error toast and navigate to `/users`.
  - Note: There is no `getApiUsersById` endpoint, so we fetch all users and filter. This is acceptable for a POC with a small user set.

- `onSubmit` handler receives `{ email, password }`:
  - Set `loading.value = true`
  - Build the body: `{ email, newPassword: password || null }` (matches `UpdateUserDto` — send null if password is empty to keep current)
  - Call `putApiUsersById({ path: { id: userId }, body: { email, newPassword: password || null } })`
  - Set `loading.value = false`
  - If `error`: `toast.add({ severity: 'error', summary: 'Error', detail: (error as any)?.detail ?? 'Failed to update user', life: 3000 })` and return
  - On success: `toast.add({ severity: 'success', summary: 'Updated', detail: 'User updated', life: 3000 })` then `router.push('/users')`

- Template:
  ```html
  <div class="p-4">
    <h1>Edit User</h1>
    <div v-if="pageLoading" class="flex justify-center p-8">
      <i class="pi pi-spinner pi-spin" style="font-size: 2rem"></i>
    </div>
    <UserForm v-else :is-create="false" :initial-email="userEmail" :loading="loading" @submit="onSubmit" />
  </div>
  ```

**Toast patterns (per CONTEXT.md locked decisions):**
- Success toasts: `severity: 'success'`, auto-dismiss `life: 3000`
- Error toasts: `severity: 'error'`, auto-dismiss `life: 3000`
- 401 errors handled by existing interceptor in main.ts — no toast needed for 401
