<task type="auto">
  <name>Task 2: Create ApplicationUser, ApplicationDbContext, and register EF Core with shared InMemoryDatabaseRoot</name>
  <files>SimpleAdmin.Api/Models/ApplicationUser.cs, SimpleAdmin.Api/Data/ApplicationDbContext.cs, SimpleAdmin.Api/Program.cs</files>
  <action>
1. Create `SimpleAdmin.Api/Models/ApplicationUser.cs`:
   - Class `ApplicationUser` extending `IdentityUser` in namespace `SimpleAdmin.Api.Models`
   - No additional properties needed for POC

2. Create `SimpleAdmin.Api/Data/ApplicationDbContext.cs`:
   - Class `ApplicationDbContext` extending `IdentityDbContext<ApplicationUser>` in namespace `SimpleAdmin.Api.Data`
   - Single constructor accepting `DbContextOptions<ApplicationDbContext>`
   - No additional DbSets needed yet (Identity + OpenIddict tables come from base class + UseOpenIddict)

3. Update `SimpleAdmin.Api/Program.cs`:
   - Add `using` statements for `Microsoft.EntityFrameworkCore`, `Microsoft.EntityFrameworkCore.Storage`, `SimpleAdmin.Api.Data`, `SimpleAdmin.Api.Models`
   - After `var builder = WebApplication.CreateBuilder(args);`, register:
     ```csharp
     var inMemoryRoot = new InMemoryDatabaseRoot();
     builder.Services.AddSingleton(inMemoryRoot);

     builder.Services.AddDbContext<ApplicationDbContext>((sp, options) =>
     {
         options.UseInMemoryDatabase("SimpleAdmin", sp.GetRequiredService<InMemoryDatabaseRoot>());
         options.UseOpenIddict();
     });
     ```
   - Register Identity per locked decision (use `AddIdentity`, NOT `AddDefaultIdentity`):
     ```csharp
     builder.Services.AddIdentity<ApplicationUser, IdentityRole>()
         .AddEntityFrameworkStores<ApplicationDbContext>()
         .AddDefaultTokenProviders();
     ```
   - Keep the minimal middleware pipeline: `app.UseRouting()`, `app.MapControllers()` (remove any Swagger/OpenAPI middleware from template)
   - Ensure `builder.Services.AddControllers()` is present

CRITICAL per locked decision: Use `AddIdentity<ApplicationUser, IdentityRole>` (NOT `AddDefaultIdentity`) to avoid default UI conflicts with OpenIddict. Pass the shared `InMemoryDatabaseRoot` singleton to `UseInMemoryDatabase()`.
  </action>

</task>
