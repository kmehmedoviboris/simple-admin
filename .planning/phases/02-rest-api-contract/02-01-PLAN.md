Create the `Dtos/` folder under `SimpleAdmin.Api/` and add three C# record files:

1. `UserListDto.cs`: `public record UserListDto(string Id, string Email, bool EmailConfirmed);` in namespace `SimpleAdmin.Api.Dtos`.
2. `CreateUserDto.cs`: `public record CreateUserDto(string Email, string Password);` in namespace `SimpleAdmin.Api.Dtos`.
3. `UpdateUserDto.cs`: `public record UpdateUserDto(string? Email, string? NewPassword);` in namespace `SimpleAdmin.Api.Dtos`. Both properties nullable — PUT can update email, password, or both.

Create `Controllers/UsersController.cs` with:
- Namespace `SimpleAdmin.Api.Controllers`
- `[ApiController]`, `[Route("api/users")]`
- `[Authorize(AuthenticationSchemes = OpenIddictValidationAspNetCoreDefaults.AuthenticationScheme)]` — use the OpenIddict validation scheme, NOT the default scheme (per Phase 1 decision)
- Constructor-inject `UserManager<ApplicationUser>`
- Four actions matching the research code examples exactly:
  - `GetAll()` — `[HttpGet]`, `[ProducesResponseType<IEnumerable<UserListDto>>(StatusCodes.Status200OK)]`, projects `_userManager.Users.Select(...).ToList()` (synchronous `.ToList()` — do NOT use `ToListAsync`)
  - `Create([FromBody] CreateUserDto dto)` — `[HttpPost]`, sets `UserName = dto.Email` and `Email = dto.Email`, `EmailConfirmed = true`, calls `CreateAsync(user, dto.Password)`, returns `CreatedAtAction(nameof(GetAll), new UserListDto(...))` on success, `BadRequest(result.Errors)` on failure
  - `Update(string id, [FromBody] UpdateUserDto dto)` — `[HttpPut("{id}")]`, `FindByIdAsync`, returns 404 if null. If `dto.Email` is not null, calls `SetEmailAsync` then `SetUserNameAsync` (CRITICAL: keep UserName in sync). If `dto.NewPassword` is not null, calls `RemovePasswordAsync` then `AddPasswordAsync` (do NOT use `ChangePasswordAsync`). Returns `Ok(new UserListDto(...))`.
  - `Delete(string id)` — `[HttpDelete("{id}")]`, `FindByIdAsync`, returns 404 if null, calls `DeleteAsync`, returns `NoContent()`.
- Add `[ProducesResponseType]` attributes for all status codes on every action (200/201/204/400/404) so OpenAPI schema generation works correctly.

Required usings: `Microsoft.AspNetCore.Authorization`, `Microsoft.AspNetCore.Identity`, `Microsoft.AspNetCore.Mvc`, `OpenIddict.Validation.AspNetCore`, `SimpleAdmin.Api.Dtos`, `SimpleAdmin.Api.Models`.
