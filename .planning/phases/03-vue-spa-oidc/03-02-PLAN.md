**Step 4: Generate the client**
The backend MUST be running for this step. Start it, generate, then stop:
```bash
# Start backend in background
cd ../SimpleAdmin.Api && dotnet run &
BACKEND_PID=$!
sleep 5  # Wait for startup

# Generate client
cd ../simple-admin-spa
npx openapi-ts

# Stop backend
kill $BACKEND_PID 2>/dev/null
```

If the backend is already running, skip start/stop. The generated files land in `src/client/`.

**Step 5: Update src/main.ts — add client configuration AFTER pinia/router/PrimeVue setup**
After `app.use(PrimeVue, ...)` and before `app.mount('#app')`, add:

```typescript
import { client } from '@/client/client.gen'

client.setConfig({ baseUrl: 'http://localhost:5000' })

client.interceptors.request.use((request) => {
  const auth = useAuthStore()
  if (auth.accessToken) {
    request.headers.set('Authorization', `Bearer ${auth.accessToken}`)
  }
  return request
})

client.interceptors.response.use(async (response) => {
  if (response.status === 401) {
    const auth = useAuthStore()
    auth.setUser(null)
    await auth.login()  // triggers signinRedirect — clears tokens and redirects
  }
  return response
})
```

Per user decisions:
- Only handle 401 (not 403) — POC has no role-based restrictions
- 401 triggers immediate redirect to login — no toast, no silent refresh
- Clear tokens and redirect — no intermediate screen

**Step 3: Create tests/auth.spec.ts**
Write these smoke tests (all cover AUTH-02 success criteria):

1. **"unauthenticated redirect"** — Navigate to `/users`. Expect the browser to end up on the Razor login page (URL contains `/Account/Login`). This proves Success Criterion 1.

2. **"login flow"** — Navigate to `/users`, get redirected to login page, fill in email `admin@simpleadmin.local` and password `Admin1234!`, submit the form. Expect to land at `/users` in the SPA. This proves Success Criterion 2.

3. **"session persistence"** — After login (reuse login helper), reload the page (`page.reload()`). Expect to remain on `/users` without being redirected to login. This proves Success Criterion 3.

4. **"logout"** — After login, click the "Logout" button. Expect to be redirected away from `/users` (back to login page or SPA root which triggers login). This proves AUTH-02 core requirement.

5. **"protected route after logout"** — After logout, navigate to `/users`. Expect to be redirected to the login page again. This proves AUTH-02.

6. **"api bearer"** — After login, intercept network requests to `/api/users`. Navigate to a page that triggers the API call (or manually invoke fetch via `page.evaluate`). Verify the request includes `Authorization: Bearer ...` header. This proves Success Criterion 5.

- After login, OpenIddict redirects to `http://localhost:5173/callback?code=...` which the CallbackView processes and redirects to `/users`
