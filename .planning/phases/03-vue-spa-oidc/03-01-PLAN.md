1. **Program.cs** — In the `.AddServer(options => ...)` block:
   - Chain `.SetLogoutEndpointUris("/connect/logout")` after `.SetTokenEndpointUris("/connect/token")`
   - Chain `.EnableLogoutEndpointPassthrough()` after `.EnableTokenEndpointPassthrough()` in the `UseAspNetCore()` block

2. **OpenIddictWorker.cs** — In the SPA client Permissions set:
   - Add the endpoint logout permission. Try `OpenIddictConstants.Permissions.Endpoints.Logout` first; if it does not compile, use `OpenIddictConstants.Permissions.Endpoints.EndSession` (OpenIddict renamed this between v6-v7; check IntelliSense). Document whichever constant works in the STATE.md decisions log.

3. **AuthorizationController.cs** — Add a new Logout action:
   ```csharp
   [HttpGet("~/connect/logout")]
   [HttpPost("~/connect/logout")]
   public async Task<IActionResult> Logout()
   {
       await HttpContext.SignOutAsync(IdentityConstants.ApplicationScheme);
       return SignOut(
           authenticationSchemes: OpenIddictServerAspNetCoreDefaults.AuthenticationScheme,
           properties: new AuthenticationProperties { RedirectUri = "/" }
       );
   }
   ```
   Add `using Microsoft.AspNetCore.Authentication;` if not already present.

   - Stop the backend after verification

Create the full Vue 3 SPA project at the repo root as `simple-admin-spa/`:

**Step 1: Scaffold with npm create vue@latest**
Run `npm create vue@latest simple-admin-spa -- --typescript --router --pinia` (non-interactive flags). If the CLI requires interactive input, select: TypeScript=Yes, JSX=No, Vue Router=Yes, Pinia=Yes, Vitest=No, E2E=No, ESLint=No, Prettier=No.

**Step 2: Install dependencies**
```bash
cd simple-admin-spa
npm install oidc-client-ts primevue @primeuix/themes primeicons
```

**Step 3: Create src/auth/oidcManager.ts**
Export a singleton `userManager` instance:
```typescript
import { UserManager, WebStorageStateStore } from 'oidc-client-ts'

export const userManager = new UserManager({
  authority: 'http://localhost:5000',
  client_id: 'simple-admin-spa',
  redirect_uri: 'http://localhost:5173/callback',
  post_logout_redirect_uri: 'http://localhost:5173/',
  response_type: 'code',
  scope: 'openid email profile api',
  userStore: new WebStorageStateStore({ store: window.sessionStorage }),
  automaticSilentRenew: false,
  loadUserInfo: false,
})
```

**Step 4: Create src/stores/authStore.ts**
Pinia store with `user` state, `isAuthenticated` and `accessToken` getters, `initialize()`, `login()`, `logout()`, and `setUser()` actions. Follow the exact pattern from 03-RESEARCH.md Pattern 2.

**Step 5: Create src/router/index.ts**
Routes:
- `/callback` → CallbackView.vue, `meta: { public: true }`
- `/oidc-error` → OidcErrorView.vue, `meta: { public: true }`
- `/users` → UsersView.vue (protected)
- `/` → redirect to `/users`

Global `beforeEach` guard: skip public routes, call `auth.initialize()` if user is null, redirect to login via `auth.login()` if not authenticated. CRITICAL: call `useAuthStore()` INSIDE the `beforeEach` callback, NOT at module top level (Pinia not yet installed at import time).

**Step 6: Create src/views/CallbackView.vue**
On mount, call `userManager.signinRedirectCallback()`, set user in authStore, redirect to `/users`. On error, redirect to `/oidc-error?reason=<message>`. Follow Pattern 4 from research.

**Step 7: Create src/views/OidcErrorView.vue**
Display the `reason` query parameter. Show a "Try again" button that calls `authStore.login()` to trigger a new signinRedirect. Per user decision: simple error page with reason and retry button.

**Step 8: Create src/views/UsersView.vue**
Placeholder page: `<h1>Users</h1><p>User management coming in Phase 4.</p>`. This route is protected by the auth guard.

**Step 9: Create src/components/AppShell.vue**
PrimeVue Toolbar (or Menubar) with:
- Left: app title "SimpleAdmin"
- Right: plain "Logout" button (PrimeVue Button component)
- No user email display (per user decision)
- No confirmation dialog (per user decision)
- Logout click calls `authStore.logout()` directly

**Step 10: Create src/App.vue**
Wrap `<router-view />` inside `<AppShell>` (show AppShell only when authenticated — check `authStore.isAuthenticated`; on public routes like /callback and /oidc-error, just show `<router-view />` without the shell).

**Step 11: Configure src/main.ts**
Registration order matters:
1. `app.use(pinia)` — FIRST (so stores are available in router guards)
2. `app.use(router)` — SECOND
3. `app.use(PrimeVue, { theme: { preset: Aura } })` — THIRD
4. Import `primeicons/primeicons.css`
5. `app.mount('#app')`

Do NOT set up the hey-api client here yet — that is Plan 03-02.

**Step 12: Clean up scaffold**
Remove default create-vue boilerplate: delete `src/components/HelloWorld.vue`, `src/components/TheWelcome.vue`, `src/components/WelcomeItem.vue`, `src/components/icons/`, `src/assets/base.css`, `src/assets/logo.svg`, `src/views/HomeView.vue`, `src/views/AboutView.vue`. Keep `src/assets/main.css` but empty it (or keep minimal reset).
